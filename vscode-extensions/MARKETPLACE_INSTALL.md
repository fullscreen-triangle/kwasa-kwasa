# 🎯 Install from VSCode Extensions Marketplace

The easiest way to get the Kwasa-Kwasa extensions is directly from the VSCode Extensions Marketplace!

## 🚀 Quick Install (30 seconds)

### Step 1: Open Extensions Tab

-   **Press `Ctrl+Shift+X`** (Windows/Linux) or **`Cmd+Shift+X`** (Mac)
-   Or click the **Extensions icon** in the Activity Bar (left sidebar)

### Step 2: Search and Install

Search for each extension and click **Install**:

1. **"Turbulance Language Support"**

    - Essential language support for `.turb` files

2. **"Knowledge Resolution Engine"**

    - Advanced knowledge validation and embedding analysis

3. **"Pattern Analysis Evidence Workbench"**

    - Pattern detection and evidence network analysis

4. **"Metacognitive Orchestration Debugger"**

    - Debug semantic processing and orchestration

5. **"Advanced Boundary Detection Studio"**
    - Multi-algorithm text boundary analysis

### Step 3: Reload VSCode

-   Click **"Reload"** when prompted, or restart VSCode
-   You'll see new icons in the Activity Bar (left sidebar)

## 🚨 Troubleshooting

### Extensions Not Appearing?

-   **Restart VSCode** completely
-   **Check Extensions panel** - ensure they're enabled (not grayed out)
-   **Update VSCode** to latest version (1.80.0+)

### No Syntax Highlighting?

-   **Save file with `.turb` or `.trb` extension**
-   **Check language mode** (bottom-right corner of editor)
-   **Manually select "Turbulance"** language if needed

### Commands Not Working?

-   **Open a `.turb` file first** (some commands require Turbulance files)
-   **Use full command names** in Command Palette
-   **Check Activity Bar** for new icons to click

## 📝 File Extensions

The extensions work with:

-   **`.turb`** - Primary Turbulance files
-   **`.trb`** - Alternative Turbulance files
-   **`.md`** - Markdown (for boundary detection)
-   **`.txt`** - Plain text (for boundary detection)
